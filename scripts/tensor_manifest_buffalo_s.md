# buffalo_s tensor-name manifest (verbatim ONNX initializers)

> Generated by `scripts/trace_onnx_manifest.py`. This is the converter
> contract: `scripts/convert_facedetect_to_gguf.py` writes every
> initializer below into the GGUF under a stable per-sub-model
> namespace prefix (`det.` / `rec.` / `ga.` / `as<i>.`), keeping the
> ONNX name otherwise **verbatim**. Every C++/ggml graph task maps the
> GGUF tensor name 1:1. Do not rename. The prefix is required because
> buffalo's SCRFD and ArcFace ONNX graphs reuse auto-numbered
> initializer names (`685`, `689`, `691`, `695`, `697`, `701`) for
> different real weights, and the C++ loader keys all tensors by name
> in one ggml context.

## Provenance

- Pack: `buffalo_s` (insightface `deepinsight/insightface` release v0.7).
- **License: non-commercial** (insightface model release terms). The
  weights are used here only to trace tensor names; the commercial-
  friendly alternative is the YuNet + SFace pack (Apache-2.0).
- Download: `buffalo_s.zip` from the insightface v0.7 release.
- Quant column reflects `--dtype f16` (the converter default).
- All source initializers are ONNX `FLOAT` (F32); quantization is
  applied by the converter on 2-D+ weights only. With the brief's
  `should_quantize`, a tensor qualifies only when **both** of its two
  innermost ggml `ne` dims are >= 32, so 3x3 / 1x1 conv kernels
  (innermost `ne` = 3 or 1) stay F32 and only large 2-D Gemm/MatMul
  weights (e.g. the ArcFace embedding `fc`) are stored F16/Q8_0.
